# Project Scorecard Template

Use this to judge whether a project is portfolio-worthy.

| Category | Questions | Score (1-5) |
|---|---|---:|
| Problem clarity | Is the problem real and easy to explain? | |
| Architecture | Does the design show good decomposition? | |
| Data grounding | Is there a clear retrieval or data strategy? | |
| Tooling | Does it interact with real systems or useful tools? | |
| Security | Are permissions, validation, and failure modes addressed? | |
| Evals | Is there a measurable test or benchmark? | |
| Observability | Can you inspect traces, latency, cost, and errors? | |
| Demo quality | Could another engineer understand it in 5 minutes? | |
| Interview story | Can you explain tradeoffs and lessons learned? | |

## Rule of thumb

If a project scores below 30 total, improve it before featuring it heavily on a resume.
